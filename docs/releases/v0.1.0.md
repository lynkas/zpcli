# v0.1.0 - First stable CLI and MCP baseline

## Highlights

- Unified `site` command group for all site management operations
- Stable MCP tool names for plugin and client integrations
- Linux release binaries for `amd64` and `arm64`
- Build metadata via `zpcli version`
- Opt-in structured logging with `-v` and `-vv`

## CLI Surface

Supported CLI commands:

- `zpcli site add <domain>`
- `zpcli site add <seriesId> <domain>`
- `zpcli site ls`
- `zpcli site rm <id>`
- `zpcli site validate`
- `zpcli site health`
- `zpcli search <keyword>`
- `zpcli detail <siteId> <vodId>`
- `zpcli <siteId> <vodId> [episode]`
- `zpcli doctor`
- `zpcli mcp`
- `zpcli version`

## MCP Surface

Supported stable MCP tools:

- `search_videos`
- `get_video_detail`
- `list_sites`
- `add_site`
- `remove_site`
- `validate_sites`
- `health_check`

## Breaking Changes

- Removed legacy CLI root-level site-management aliases
- Removed legacy MCP aliases `search` and `get_detail`

## Linux Artifacts

This release publishes:

- `zpcli_linux_amd64`
- `zpcli_linux_arm64`
- `checksums.txt`

## Upgrade Notes

- Update automation and plugins to use `zpcli site ...`
- Update MCP clients to use `search_videos` and `get_video_detail`
- If debugging binary compatibility, run `zpcli version`
