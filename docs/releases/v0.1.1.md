# v0.1.1 - Release pipeline compatibility fix

## Highlights

- Fixes the GitHub Release workflow after introducing `log/slog`
- Declares Go `1.21` as the minimum supported Go version

## What Changed

- Updated the project Go version requirement to `1.21`
- Ensured release builds and CI use a compatible Go toolchain through `go.mod`

## Why This Release Exists

- The original `v0.1.0` release flow failed because `log/slog` is not available in Go `1.19`
- `v0.1.1` is the first Linux release that should complete successfully through GitHub Actions

## Linux Artifacts

This release publishes:

- `zpcli_linux_amd64`
- `zpcli_linux_arm64`
- `checksums.txt`

## Upgrade Notes

- If you are integrating with an AstrBot plugin, use `v0.1.1` or later
- If you build from source, use Go `1.21` or newer
