# v0.1.2 - MCP search pagination guidance

## Highlights

- Adds pagination support to the MCP `search_videos` tool through an optional `page` argument
- Clarifies MCP client search behavior so agents page first and avoid expanding short core keywords

## What Changed

- Updated the MCP `search_videos` schema, tool description, and runtime handling to accept `page`
- Added no-result guidance that tells clients to retry later pages with the same keyword
- Updated MCP documentation and OpenClaw integration guidance for the new search behavior
- Aligned MCP server initialization metadata with the build version instead of a hardcoded value

## Upgrade Notes

- MCP clients can keep calling `search_videos` with only `keyword`; `page` is optional
- If page `1` does not contain the expected result, retry with later pages before changing the keyword
- Avoid expanding a short core keyword with extra words during MCP search retries
